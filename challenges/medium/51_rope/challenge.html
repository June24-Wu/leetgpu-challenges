Implement a single head Rotary Position Embedding (RoPE) forward pass under single batch. Given an input tensor of shape [S, H] where:

  • S = sequence length
  • H = hidden dimension (must be even)

Given an base for geometric progression used to compute the rotation angles. RoPE applies a position-dependent rotation to pairs of features:

\[ \begin{align} \text{pos} &= [0, 1, \dots, S-1]^\top \\ \text{freq}_i &= \text{base}^{-\frac{2i}{H}}, \quad i = 0, 1, \dots, \tfrac{H}{2}-1 \\ \theta_{s,i} &= \text{pos}_s \cdot \text{freq}_i \\ \cos\theta_{s,i}, \; \sin\theta_{s,i} & \quad \text{are applied to each pair of features} \\ x_{s,2i}, x_{s,2i+1} &= \text{even and odd indices of input vector at position } s \\ y_{s,2i} &= x_{s,2i}\cos\theta_{s,i} - x_{s,2i+1}\sin\theta_{s,i} \\ y_{s,2i+1} &= x_{s,2i}\sin\theta_{s,i} + x_{s,2i+1}\cos\theta_{s,i} \end{align} \]

Implementation Requirements

  • Use only native features (external libraries are not permitted)
  • The solve function signature must remain unchanged
  • The final result must be stored in the output tensor

Example 1:

Input:  
  input = [[1.0, 0.0],
           [0.0, 1.0],
           [1.0, 1.0]]   (S=3, H=2)
  base = 10000.0

Output: 
  [[ 1.0000000, 0.0000000],
   [-0.8414710, 0.5403023],
   [-1.3254442, 0.49315065]]

Example 2:

Input:  
  input = [[5.0, 3.0]]   (S=1, H=2)
  base = 10000.0

Output: 
  [[5.0000000, 3.0000000]]

Constraints

  • 1 ≤ S ≤ 10,000
  • H is even
  • -100.0 ≤ input values ≤ 100.0
  • 1.0 ≤ base ≤ 10,000.0
